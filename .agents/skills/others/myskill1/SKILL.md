## 核心能力 (Core Capabilities)

*   **資料庫存取與解析**：能夠查詢 SQLite 資料庫 (`chat.db`) 中的 `ChatSession` 與 `ChatMessage`，精準過濾出帶有 `tool` 角色或關鍵字 (如「系統自動抽牌結果」) 的抽牌歷史。
*   **趨勢統整與報告**：能將凌亂的對話紀錄轉化為結構化的統計報表。例如：計算特定卡牌出現的頻率、正逆位的比例，或彙整過去一週的占卜主軸。
*   **匯出與備份**：能協助使用者將歷史抽卡結果打包匯出成 CSV、JSON 等格式，方便跨平台儲存。

## 執行步驟
當使用者要求「幫我整理抽卡紀錄」或輸入對應指令時，請遵循以下步驟：

### 1. 讀取並解析資料庫 (Fetch Data)
* 使用 Python 腳本連線至專案底下的 `instance/chat.db`。
* 撈取 `ChatMessage` 表格中，`role='tool'` 的紀錄，並對照同一個 `session_id` 中的使用者問題。

### 2. 結構化分析 (Analyze & Structure)
* 統計過去一段時間內出現過的塔羅牌及其狀態 (正位/逆位)。
* 將抽牌時間 (`created_at`)、使用者詢問的問題與抽卡結果對應起來。

### 3. 產出報告 (Generate Report)
* 根據使用者的要求，產出對應格式的結果：
  * **文字總結**：例如「你過去五次占卜中最常出現的牌是『愚者』，這顯示了...」。
  * **表格呈現**：使用 Markdown 表格列出詳細的時間、問題與抽到的牌。
  * **檔案匯出**：如果使用者要求備份，自動撰寫一段腳本將結果輸出為 `tarot_history.csv` 並存放在專案根目錄。

## 輸出要求
* 報告必須清晰易讀，盡量運用 Markdown 表格呈現歷史數據。
* 在給予總結分析時，請保持客觀且帶有一點神祕學的專業洞察語氣。
